mklink 命令的使用

发表于 2019-08-21 更新于 2019-12-28 分类于 Computer Skills Valine：

介绍

mklink 命令是 Windows

使用场景

OneDrive 同步盘

同步非OneDrive文件夹下的文件，比如EndNote的数据库文件
当然，同样适用于坚果云/Dropbox/Google Drive 等同步盘

非系统盘部署安装 Office 2019/365

Office 2019及之后采取了部署安装，有别于之间的Office 2003/2007/2013/2016，不提供离线安装，更没有UI来进行选择安装组件和安装位置，通过部署工具安装

隐藏私密文件

使用 U 盘解锁，但是当文件过大时，采用disk检索可能会被发现，这一点还没有验证

mklink 连接的取消

使用rmdir取消。

删除虚拟的链接目录，并不会删除远程文件夹真实文件，注意千万不能用del，del会删除远程的真实文件

Chrome浏览器书签备份

发表于 2019-03-02 更新于 2020-02-08 分类于 Computer Skills Valine：

实现方案

对于普通文件的单纯备份，往往一个硬盘了事，有可靠的网盘的话就更好了；对于一些频繁更改的文件比如我们的大论文啊，就需要版本控制了，Git是个不错的选择，当然有一定的学习成本了，而且现在一些网盘也有这些功能了。

而对于浏览器中的数据，比如书签、密码、浏览记录等等，这些文件内容天天改动，有时候跨平台、跨机器想找个网页还真挺麻烦……FireFox、Chrome这些浏览器都有备份功能，一来我自己不偏好网络备份敏感数据（虽然他不值钱），二来这些备份需要一直联网（这一点偶尔会不大友好，突然联网后书签栏会炸掉）。此外，Xmarks也是个第三方多浏览器同步备份的工具，可惜挂了。

然而手动保存太废事了，，关键是脑子不好人又懒，往往记不得去备份……于是想能不能自动导出到同步盘呢？说是实现方案，其实是做一回脚本小子，写个批处理使用现有的方案将导出的HTML文件保存到同步盘。
